## 2. 背景与目标

### 2.1 业务背景

Claude Code 作为命令行 AI 助手，目前只能在 Mac 终端内交互。开发者的典型痛点：

- **离开桌面就断联**：在 Mac 上让 Claude Code 执行长任务（编译、爬取、深度对话）后离开座位，无法查看进度、无法干预
- **多任务并行困难**：同时跑多个 Claude Code 会话（不同项目）时，终端 Tab 管理混乱，恢复历史对话繁琐
- **移动场景无入口**：通勤、出差时想继续推进 Claude Code 的对话，没有合适的客户端
- **远程批准卡壳**：Claude Code 需要 tool_use 批准时，离开 Mac 就只能等回到桌面，常常导致任务停滞

现有工具（Termius、Blink Shell、tmate）都不能完整满足"集中管理 + 跨端 + Claude Code 集成"的诉求。

### 2.2 项目目标

打造一个**个人专属、轻量级**的跨端 Claude Code 协作客户端，让开发者：

1. 在 Mac 上以多 Tab 方式集中管理所有 Claude Code 会话（每 Tab 绑定一个本地项目目录）
2. 通过 Android 手机实时查看任意 Tab 的对话进展，无需 SSH
3. 在 Android 上以卡片消息形式发送文字、图片继续与 Claude Code 对话
4. 在 Android 上批准/拒绝 Claude Code 的 tool_use 请求

### 2.3 成功指标

| 指标 | 目标值 |
|------|--------|
| Mac 端启动到拉起所有 Tab | < 3 秒（10 个 Tab 以内）|
| 单条消息从 Mac → 手机端可见 | < 500ms（同地域 VPS）|
| 手机端进入 Tab 首屏渲染 | < 1 秒（50 条历史）|
| 客户端崩溃率 | < 0.5% |
| Tab 列表 + 对话状态恢复成功率（软件重启后）| 100%（依赖 `claude -c` 自动恢复）|

## 3. 用户故事

### 3.1 角色定义

本项目为**个人工具**，只有一个角色：**Owner（设备所有者）**。Owner 持有一台 Mac、一台或多台 Android 手机，所有设备归同一人所有。

### 3.2 核心用户故事

| ID | 作为... | 我希望... | 这样我就能... |
|----|---------|----------|--------------|
| US-01 | Owner | 在 Mac 上一次性看到所有 Claude Code 会话 | 不用在多个终端 Tab 间切来切去 |
| US-02 | Owner | 软件重启后所有 Tab 自动恢复历史对话 | 不丢失上下文，不重复手动 `claude -c` |
| US-03 | Owner | 在手机上看到 Mac 上正在跑的 Claude Code 的最新进展 | 离开桌面也能掌握任务状态 |
| US-04 | Owner | 在手机上输入文字让 Claude Code 继续 | 路上灵感来了立刻能补一句 |
| US-05 | Owner | 在手机上发图给 Claude Code 解读 | 看到截图想分析，不用回家 |
| US-06 | Owner | 在手机上批准 Claude Code 的 tool_use | 长任务不会因为没人批准就停滞 |
| US-07 | Owner | 绑定多台手机 | 工作机 + 个人机分别都能远程接入 |
| US-08 | Owner | 撤销某台手机的绑定 | 手机丢了能远程剔除 |
| US-09 | Owner | 看到 Mac/Server/Claude 进程的运行状态 | 出问题时能快速判断是哪一层 |
| US-10 | Owner | Mac 离线时手机端有明确提示 | 不至于以为消息丢了 |

### 4.2 功能范围边界

#### ✅ 做（in scope）

- 一个 Owner、一台 Mac、N 台 Android（N ≥ 1）
- Mac 端原生 Swift（macOS 14+）、Android 端 Flutter（Android 10+）
- Server 部署在 Owner 自有 VPS（Linux + Docker）
- Claude Code 通过 `claude -c` 调用，依赖 Claude Code 本身的对话持久化
- 文字 + 图片输入；tool_use 批准/拒绝/总是批准
- 仅 TLS 加密，自签或 Let's Encrypt 证书
- 三端独立本地日志 + 查看/导出入口

#### ❌ 不做（out of scope）

- 多 Owner / 多租户 / 用户系统
- iOS / Windows / Linux 客户端
- 端到端加密、密钥协商
- 完整 TUI 模拟器（手机端纯卡片消息）
- 离线消息持久化队列（Server 完全无状态）
- 团队协作 / 多人共享 Tab
- Mac 后台 daemon / 开机自启
- 同一文件夹多 Tab 并行
- 自动化测试套件（本项目阶段一末尾用户选择"不需要测试用例和自动化测试"）

## 5. 非功能性需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| MacClient 内存占用（10 Tab）| < 500 MB |
| AndroidClient 内存占用 | < 200 MB |
| Server 内存占用（5 设备连接）| < 50 MB |
| 单 WebSocket 消息往返延迟 | < 200ms（同地域）|
| JSONL → 手机端展示总延迟 | < 500ms（含 100ms 节流）|
| 手机端 50 条消息滚动 FPS | ≥ 50 fps |

### 5.2 安全

- 所有 WebSocket 连接强制 TLS（自签证书时 MacClient/AndroidClient 内置信任）
- 主 token 与 sub_token 均为 ≥ 32 字节随机串
- Server 端 token 哈希存储，不明文落盘
- 手机端发图先经 Server 中转，**Mac 拉取后立即从 Server 删除**（防止图片堆积）
- MacClient 配置文件 (`tabs.json` / token 文件) 存储到 `~/Library/Application Support/cc-anywhere/`，权限 `0600`

### 5.3 兼容性

| 端 | 最低版本 |
|----|---------|
| MacClient | macOS 14 (Sonoma) |
| AndroidClient | Android 10 (API 29) |
| Server 运行环境 | Linux + Docker 20.10+ |
| Claude Code | 与 JSONL 格式兼容的所有版本（首次启动探测版本，发现不兼容提示用户）|

### 5.4 可用性

- MacClient 启动到能用：< 3 秒
- AndroidClient 扫码到绑定成功：< 5 秒
- 连接断开自动重连：间隔 1s/3s/10s/30s 指数退避
- 软件崩溃自动清理子进程：MacClient 注册 `atexit` 钩子

### 5.5 可维护性

- 三端代码独立目录，各自有 README
- 协议消息有版本字段，未来兼容升级
- 关键操作有 INFO 级别日志（连接建立、Tab 创建、消息收发、错误）

## 7. 风险与依赖

### 7.1 已知风险

| ID | 风险 | 影响 | 缓解方案 |
|----|------|------|---------|
| R-01 | Claude Code JSONL 格式跨版本变化 | 高 - 手机端消息解析失效 | 启动时探测 JSONL 字段集合，识别已知版本；版本不识别时 fallback 显示原始 JSON + 提示升级 |
| R-02 | FSEvents 在 macOS Sandbox 下需要授权 | 中 - 首次使用受阻 | 用 Apple Document-scoped bookmarks，首次启动引导用户授权 `~/.claude/projects/` |
| R-03 | 公网穿透延迟波动 | 中 - 实时性变差 | Server 部署在地理位置近的 VPS；UI 显示延迟标识；批处理参数可调 |
| R-04 | 软件崩溃时子进程残留 | 中 - PTY/claude 进程泄漏 | atexit 钩子 + 进程组 SIGKILL 兜底 + 启动时扫描历史 PID 清理 |
| R-05 | 自签 TLS 证书的客户端信任繁琐 | 低 - 首次部署体验差 | 默认引导使用 Let's Encrypt + ACME-DNS（无需 80/443）；自签作为 fallback |
| R-06 | 手机端图片注入路径权限 | 低 - 图片读取失败 | Mac 端固定使用 `~/Library/Application Support/cc-anywhere/inbox/`，路径稳定且 App 拥有写权限 |
| R-07 | 多手机同时输入冲突 | 低 - 个人工具单人使用为主 | 已决策不做应用层控制，全塞 PTY 自然顺序处理 |

### 7.2 外部依赖

| 依赖 | 类型 | 风险等级 |
|------|------|---------|
| Claude Code CLI | 运行依赖 | 高（如果 Anthropic 大改命令行或 JSONL 格式）|
| SwiftTerm | Swift 包依赖 | 低（活跃维护）|
| Flutter SDK | 构建依赖 | 低 |
| Go 标准库 + `gorilla/websocket` 或 `nhooyr.io/websocket` | 构建依赖 | 低 |
| Owner 自有 VPS | 运行依赖 | 中（VPS 宕机则手机端不可用）|

### 7.3 降级方案

| 场景 | 降级行为 |
|------|---------|
| Server 不可达 | MacClient 仍可继续本地使用，状态灯转红，本地消息缓存，恢复后批量同步当前状态（不补发历史增量）|
| Mac 离线 | AndroidClient 顶部显示"Mac 离线"，所有操作禁用 |
| Claude Code 进程崩溃 | Tab 内显示退出码 + "重启" 按钮，用户点击后重新 `claude -c` |
| JSONL 解析失败 | 该消息以原始 JSON 字符串显示，标记"无法解析"，不阻塞其他消息 |

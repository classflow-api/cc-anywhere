### 🟡-3 unwatch / unwatchAll 不清理 activeSubAgents / pendingMatches — 已修复

**初审定位**：`JSONLWatcher.swift` 的 `unwatch(tabId:)` / `unwatchAll()` 仅停止 stream + 移除 streams[tabId]，未清理 `activeSubAgents` / `pendingTaskMatches` / `pendingAgentMatches` → 长期开关 tab 累积内存（量级不致命但属"资源管理不闭环"）。

**修复实证**（`JSONLWatcher.swift:345-367`）：
```swift
public func unwatch(tabId: UUID) {
    guard let s = streams.removeValue(forKey: tabId) else { return }
    s.stop()
    // 第三轮 review 🟡-3：tab 关闭时清理 sub-agent 索引，避免长期累积。
    // 在 queue 上同步执行 — activeSubAgents / pendingTaskMatches /
    // pendingAgentMatches 的所有写都串行化在此 queue。
    queue.sync {
        activeSubAgents.removeValue(forKey: tabId)
        pendingTaskMatches.removeValue(forKey: tabId)
        pendingAgentMatches.removeValue(forKey: tabId)
    }
}

public func unwatchAll() {
    for s in streams.values { s.stop() }
    streams.removeAll()
    // 同 unwatch：清理所有 tab 的 sub-agent 索引
    queue.sync {
        activeSubAgents.removeAll()
        pendingTaskMatches.removeAll()
        pendingAgentMatches.removeAll()
    }
}
```

**关键风险（queue.sync 死锁）独立验证**：

unwatch 内 `queue.sync` 会在 watcher.queue 上同步执行清理；若有任何调用栈处于 watcher.queue 上下文，会触发自我死锁。复审追踪所有 unwatch 调用方：

| 调用栈 | 调用方位置 | 调用线程 | 死锁风险 |
|--------|----------|---------|---------|
| `watch(tab:)` line 82 | JSONLWatcher 内 | `watch` 由 DependencyContainer.tabs sink 调（Combine MainActor）| ✅ 无 |
| `unwatchAllExcept(ids:)` line 377 → `unwatch` line 378 | JSONLWatcher 内 | 同上（Combine sink）| ✅ 无 |
| `DependencyContainer:125` `jsonlWatcher.unwatchAllExcept` | tabs.sink Combine | MainActor | ✅ 无 |
| `DependencyContainer:258` `jsonlWatcher.unwatchAll()` | `appWillTerminate` | MainActor | ✅ 无 |
| WatchStream 内部 throttle / FSEvent callback | 不会调 unwatch | watcher.queue | ✅ 无（不调用） |

所有 unwatch 调用栈均在 MainActor 或 Combine sink 上（**非 watcher.queue**），`queue.sync` 进入 queue 同步执行清理后返回 — **无死锁路径**。与既有 `findSubAgent` (line 320-323) / `parentToolUseId(tabId:agentId:)` (line 327-330) 同 `queue.sync` 跨线程访问模式一致。

**资源管理完整性核对**：

| 字段 | unwatch 清理？ | unwatchAll 清理？ |
|------|---------------|------------------|
| `streams[tabId]` | ✅ removeValue | ✅ removeAll |
| `activeSubAgents[tabId]` | ✅ queue.sync removeValue | ✅ queue.sync removeAll |
| `pendingTaskMatches[tabId]` | ✅ queue.sync removeValue | ✅ queue.sync removeAll |
| `pendingAgentMatches[tabId]` | ✅ queue.sync removeValue | ✅ queue.sync removeAll |
| `hookPushedToolUseIds` | ⚠️ 未清理（10 分钟 TTL 自动清，全局共享，可接受 — 见 🟢-6） | ⚠️ 未清理（同左） |

`hookPushedToolUseIds` 是全局 `[String: Date]` 字典（按 toolUseId 索引，与 tabId 无关），属于"按 toolUseId 自然过期"的设计；不在 🟡-3 处置范围内，未引入新问题。

**复审判定**：处置 A（已修复）。清单 21 "关闭 watch tab 时 activeSubAgents[tabId] 是否清理（避免内存泄漏）" 现在满足；queue.sync 死锁风险已独立验证不存在。

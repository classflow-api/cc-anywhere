# 工作区权限模式 — 变更说明

| 文档项 | 内容 |
|---|---|
| 流程级别 | L3 - 小型变更 |
| 版本 | v1.0 |
| 日期 | 2026-05-18 |
| 涉及端 | Mac（Android / Server 零改动） |

## 1. 变更内容

允许用户在创建工作区时为该工作区选择 Claude Code 的 **permission mode**（权限模式），并能在创建后随时修改。

Claude CLI 通过 `--permission-mode <mode>` flag 提供这一能力，支持的值（`claude --help` 实测）：

| mode | 行为 | 适用场景 |
|---|---|---|
| `default` | 仅读取 | 入门、敏感工作 |
| `acceptEdits` | 读取 + 文件编辑 + 常见文件系统命令（mkdir/touch/mv/cp 等） | 迭代审查中的代码 |
| `plan` | 仅读取 | 修改前的代码库探索 |
| `auto` | 所有操作 + 后台安全检查 | 长时间任务、减轻提示疲劳 |
| `dontAsk` | 仅预先批准的工具 | 锁定的 CI / 脚本 |
| `bypassPermissions` | 所有操作 + 后台安全检查 | 仅隔离容器 / VM |

### 1.1 创建工作区流程

1. 用户点 **侧栏 + 按钮** 或 **顶部 Tab Strip + 按钮** → NSOpenPanel 选 folder
2. 选完 folder 后弹一个 NSAlert：
   - 标题：「选择权限模式」
   - accessoryView：NSPopUpButton（6 个选项 + 每项后跟简短说明文本）
   - "确定" / "取消" 按钮
3. 用户确定后 → 创建 Tab（`Tab.permissionMode = 选中值`）→ ProcessHost 启动 claude 子进程并传 `--permission-mode <选中值>`
4. 取消 → 不创建 Tab，回到原状态

默认选中：`default`。

### 1.2 创建后修改

- 侧栏 Tab 右键菜单新增「**权限模式**」子菜单，列出 6 个 mode（当前模式打勾）
- 用户选择不同 mode → 弹 NSAlert 提示「修改权限模式会重启 Claude 子进程（对话历史会自动保存）。继续？」
- 用户确认 → `Tab.permissionMode` 更新、`ProcessHost.stopProcess(tab.id)` → `ProcessHost.startProcess(for: tab)` 重启
- 用户取消 → 不变

## 2. 变更原因

Claude Code 不同 mode 适合不同场景，固定 `default` 对"代码改造"场景过于繁琐；固定 `acceptEdits`/`auto` 对"安全敏感"工作不合适。让用户按工作区选择，是最自然的颗粒度（同一项目通常长期用一种 mode）。

## 3. 影响范围

### 3.1 改动文件清单

| 文件 | 改动 |
|---|---|
| `MacClient/Sources/CCAnywhere/Models/Tab.swift` | Tab struct 新增 `permissionMode: String = "default"` 字段；纳入 Codable 持久化（key: `permission_mode`，反序列化时缺失 → "default"） |
| `MacClient/Sources/CCAnywhere/Services/TabManager.swift` | `createTab` 加 `permissionMode: String = "default"` 参数；新增 `setPermissionMode(_ id: UUID, _ mode: String)` 方法（触发 `.renamed` 复用事件或新增 `.permissionModeChanged`，简单起见用 `.renamed` 同样能驱动 sync） |
| `MacClient/Sources/CCAnywhere/Services/ProcessHost.swift` | `startProcess` 中拼接 args：`args = (hasHistory ? ["-c"] : []) + ["--permission-mode", tab.permissionMode]` |
| `MacClient/Sources/CCAnywhere/Views/SidebarView.swift` | `createTab` 流程加 `selectPermissionMode()` 对话框；contextMenu 加「权限模式」子菜单 + `changePermissionMode(tab, to:)` 方法 |
| `MacClient/Sources/CCAnywhere/Views/TabStripView.swift` | 同上（保持两端入口一致） |
| `MacClient/Sources/CCAnywhere/Models/PermissionMode.swift`（新建） | 定义 `enum PermissionMode: String, CaseIterable` 包含 6 个 case + `displayName: String` + `description: String`；提供 `PermissionMode.allDisplayLabels()` 工厂 |
| `AndroidClient/` | **零改动**（无需感知 mode；如未来要在手机端显示当前 mode，扩展协议即可） |
| `Server/` | **零改动** |

### 3.2 兼容性

- **协议**：Tab 持久化的 JSON 兼容旧版本：缺失 `permission_mode` 字段时反序列化为 `"default"`
- **行为变化**：原有 Tab 升级后自动按 `default` 模式启动；用户可随时改
- **回滚**：删除新字段不破坏旧持久化数据（多余的 key 被 Codable 忽略）

## 4. 验证方式

| 场景 | 步骤 | 期望 |
|---|---|---|
| 创建 + default | 新建 Tab → 选 folder → 选 default → 确定 | claude 子进程参数含 `--permission-mode default`；UI 创建成功 |
| 创建 + acceptEdits | 同上选 acceptEdits | 参数含 `--permission-mode acceptEdits` |
| 取消创建 | 选 folder 后取消 | 不创建 Tab，回到主界面 |
| 旧 Tab 升级 | 重启 App（旧持久化无 permissionMode 字段） | 旧 Tab 以 `default` 模式启动 |
| 修改模式 | 右键 Tab → 权限模式 → 选 auto → 确认 | claude 子进程被杀；以 `--permission-mode auto -c` 重启；对话历史保留 |
| 取消修改 | 同上但点取消 | Tab 不重启 |
| 重启不丢 | 修改模式后退出 App 重启 | 新 mode 持久化保留 |

## 5. 非目标 / 不做的事

- Android 端**不**显示当前 mode（侧栏 / Tab 卡片不变）
- 不做"全局默认 mode"配置（每个 Tab 自己持有）
- 不做 mode 切换的二次确认弹窗序列（一次确认即够）
- 不做 mode 历史回放（直接覆盖）
- 不影响 hook 机制 / JSONL 监听 / ws 同步等已有行为

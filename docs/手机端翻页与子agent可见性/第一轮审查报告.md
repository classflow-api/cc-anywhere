**初审定位**：HistoryBridge.readHistory 是 nonisolated public static，直接读 JSONL → AnyJSON 数组，没有走 MsgStreamBridge.injectParentToolUseId 等价路径 → 历史子 agent 在 phone 端走 agentId fallback 与 Task tool_use 主流块永远不合并。

**修复实证**（`HistoryBridge.swift:71-128`）：
- `readHistory` 内部就地实现两阶段匹配：先 `buildAgentIdToParentMap(parsed:)` 扫一遍 parsed 列表，分别建立 `parentByHash` / `agentByHash`，hash join 得到 agentId → parentToolUseId 字典；然后对每条 sidechain 记录调 `injectParentToolUseId(any:agentMap:)` 注入 `parent_tool_use_id` 字段。

### 🟡-2 silentInitialScan 与 FSEvent 跨线程数据竞争 — 已修复

**初审定位**：silentInitialScan 在 MainActor 上同步读 + mutate `seenUuids` / `fileOffsets` / `activeSubAgents` / `pendingTaskMatches` / `pendingAgentMatches`，而 FSEvent callback 已经在 watcher.queue 上并发 mutate 同一集合 → 数据竞争，潜在 EXC_BAD_ACCESS。

**修复实证**（`JSONLWatcher.swift:483-491`）：
```swift
identifyActiveSession()
queue.async { [weak self] in
    self?.silentInitialScan()
}
```
- silentInitialScan 现在 dispatch 到 watcher.queue 上执行（**串行**），与 FSEvent callback 共用同一 serial queue 排队。

**剩余 race window 分析**：
- `FSEventStreamStart(s)` 在 line 472 已启动 FSEvent dispatch 到 queue。
- 当前线程（MainActor）继续执行到 line 489 `queue.async {...}` 入队 silentInitialScan。
- 极短窗口内（µs 级），如果 FSEvent 在 line 472 - 489 之间触发并先入队，则 handleEvent 先执行，silentInitialScan 后执行。
- handleEvent 走 readNewLines 的代码已经用 `fileOffsets[file] ?? 0` 兜底 → 即使 silentInitialScan 后续运行覆写 `fileOffsets[file] = data.count`，导致重读，`seenUuids` 还会去重 → 行为退化为"重复处理一遍 indexMatchingHints"，是**等幂操作**：registerMatch 同输入同输出，onAgentFirstMessage 在🟡-3 修复后不会覆盖已正确写入的 meta。
- 关键点：**所有写都在同一 serial queue 上，单线程访问 Dictionary，无 EXC_BAD_ACCESS**。

**复审判定**：处置 A（已修复）。竞争消除（serial queue 串行化），残留只是潜在的双扫描等幂操作，不致功能错误。

### 🟢-2 HistoryBridge.promptHash 注释与实现不符（新增）

`HistoryBridge.swift:204-210` 的 `promptHash` 注释声称 "SHA1 前 200 字符（与 JSONLWatcher.promptHash 一致）"，但实现是 Swift `Hasher` 不是 SHA1：
```swift
nonisolated private static func promptHash(_ prompt: String) -> String {
    let truncated = String(prompt.prefix(200))
    var hasher = Hasher()
    hasher.combine(truncated)
    return String(hasher.finalize(), radix: 16, uppercase: false)
}
```
功能正确（HistoryBridge 内部自洽 join），仅注释误导。建议把注释改为 "与 JSONLWatcher 同语义（按 prompt 前 200 字符做哈希匹配），实现细节独立 — 内部 join 不需要跨 actor 共享 hash 值"。属于主观风格建议，不阻塞。

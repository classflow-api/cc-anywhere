# 工作区改名功能 — 变更说明

| 文档项 | 内容 |
|---|---|
| 流程级别 | L3 - 小型变更 |
| 版本 | v1.1 |
| 编写日期 | 2026-05-18 |
| 涉及端 | Mac（Android 零改动，Server 不变，dumb proxy） |

## 1. 变更内容

让用户可以从 Mac 客户端给 Tab（工作区）自定义显示名，并同步显示到手机端。

**采用复用现有字段策略（v1.1 重设计）：**

- **不新增字段**。Tab 数据模型现有 `name: String` 字段语义扩展为"显示名"（创建时默认 = folder 末尾段；可被用户改写）。
- Mac 端：
  - **侧栏（SidebarView）** Tab 行的右键菜单新增"重命名…"项（原已存在的水平 **TabStripView** 右键菜单已有此项，保持不变）。
  - 点击"重命名…"弹出 `NSAlert + NSTextField` 输入框，预填当前 name；确定提交、取消保持原样。
- 提交规则（统一更新到 `TabManager.renameTab(_:to:)`）：
  - 输入串先 `trim()` 去前后空白。
  - trim 后非空：写入 `tab.name`；**长度上限改为 40 字符**，超长截断（不抛错）。
  - trim 后为空：视为"重置为默认"，`tab.name = folder.lastPathComponent`。
- UI 显示：所有显示位置（Mac 侧栏、Mac TabStrip、Mac 内容标题、Android 列表、Android 聊天 AppBar）继续使用 `tab.name`，无需调整。
- Android 端：**零代码改动**。当前 `TabInfo.name` 字段已通过 `tab.changed` 协议接收并刷新 UI，Mac 改名后会自动同步过去。

## 2. 变更原因

工作区默认显示 folder 末尾段，多项目同名 folder 易混淆（如 `/path/A/server` 与 `/path/B/server` 都显示 `server`）。允许自定义名后，用户可起业务化名称（如 "BJYQ 后端"、"舆情新版"），手机端尤其受益。

技术上，项目本已存在 `TabManager.renameTab` 方法和水平 TabStripView 的右键改名入口，但用户日常使用的是侧栏 SidebarView，那里没有改名入口。本次主要补齐侧栏入口，并对改名规则做体验优化（trim、空回退默认、长度 40）。

## 3. 影响范围

### 3.1 改动文件清单

| 文件 | 改动 |
|---|---|
| `MacClient/Sources/CCAnywhere/Services/TabManager.swift` | `renameTab(_:to:)`：trim 输入；非空 → 截断 40 字符；空 → 用 folder.lastPathComponent 重置。`TabError.nameTooLong` 不再抛出（保留枚举不影响调用） |
| `MacClient/Sources/CCAnywhere/Views/SidebarView.swift` | contextMenu 加 "重命名…" 项；新增 `startRename(_ tab: Tab)` 方法（NSAlert + NSTextField 风格与 TabStripView 一致） |
| `MacClient/Sources/CCAnywhere/Views/TabStripView.swift` | 已有的 startRename 不变（renameTab 行为升级后自动受益于 trim/空回退） |
| `AndroidClient/` | **零改动**（`TabInfo.name` 已存在并参与同步） |
| `Server/` | **零改动**（dumb proxy） |

### 3.2 兼容性

- **协议**：`tab.changed` 协议中 `name` 字段已存在，旧版本客户端无感知变化。
- **本地持久化**：Tab.name 一直是 String（非 Optional），无序列化兼容问题。
- **行为变化**：原 `renameTab` 不接受空字符串（抛 nameTooLong 也不抛空错），现行为变为"空字符串 = 重置为默认"；长度上限从 50 → 40。两者都是更宽松/更友好的行为，无破坏性。

## 4. 验证方式

| 场景 | 步骤 | 期望结果 |
|---|---|---|
| Mac 侧栏右键改名 | 右键侧栏 Tab → 重命名… → 输入"BJYQ 后端" → 确定 | 侧栏 / TabStrip / 聊天页标题立即变为"BJYQ 后端" |
| 同步到 Android | Mac 改名后观察手机端 | 手机端 Tab 列表 ≤2s 内显示新名 |
| 清空回退默认 | Mac 重命名输入空 / 仅空格 → 确定 | name 重置为 folder 末尾段 |
| 超长截断 | Mac 输入 100 字符 | 实际存储/显示前 40 字符 |
| Emoji / 中文 | Mac 输入"🎉 测试" | 完整显示，Android 同步 |
| 取消保留 | 弹框点取消 | 名称不变 |
| 重启不丢 | Mac 改名后退出 App 再启动 | 自定义名持久化保留 |
| TabStrip 入口仍可用 | TabStripView 右键 → 重命名… | 与侧栏入口行为一致 |

## 5. 非目标 / 不做的事

- Android 端**不**提供主动改名入口（用户决策）。
- 不引入新字段、不新增协议类型。
- 不做名称查重 / 唯一性校验。
- 不做改名历史 / 撤销。
- 不影响 Tab 对应的 cwd 和 Claude SDK 数据存储位置。

## 需求清单

### Mac 端修复(9 个)

#### 需求 1:Preferences 输入框无法编辑
- **类型:fix**
- **根因:** 命令行直接运行裸 binary 时 Launch Services 默认注册为 `.accessory`(背景进程),SwiftUI 窗口可见但无法成为 key window,所有 TextField 都收不到键盘事件。
- **方案:** `AppDelegate.applicationDidFinishLaunching` 显式调 `NSApp.setActivationPolicy(.regular)` + `NSApp.activate(ignoringOtherApps:)`。
- **修改文件:** `MacClient/Sources/CCAnywhere/App/AppDelegate.swift`(+5 行)

#### 需求 2:WSClient 改地址后重连风暴
- **类型:fix**
- **根因:** `connect()` 主动 `cancel(.goingAway = 1001)` 旧 task,触发 didCloseWith → handleDisconnect 安排新 reconnect,与已发起的新连接形成无限循环;`receive` 回调同理。
- **方案:** 引入 `intentionalCancel` 标记,主动 cancel 前置位;`didCloseWith` 看到 true 直接 return + reset;receive 回调用 `t === self.task` stale task 比对,忽略旧 task 触发的回调。
- **修改文件:** `MacClient/Sources/CCAnywhere/Services/WSClient.swift`

#### 需求 7:WSClient 心跳 pong 超时检测
- **类型:fix**
- **根因:** 心跳只发 ping,**没有 pong 超时检测**。Server 不回 pong 时 socket 表面 ESTABLISHED 但实际死亡,Mac 客户端不自知。
- **方案:** 5s ping × 3 次未收到 pong → 主动 `handleDisconnect("心跳超时")` 触发 backoff 重连。
- **修改文件:** `MacClient/Sources/CCAnywhere/Services/WSClient.swift`

#### 需求 13:ImageDownloader 不信任自签证书
- **类型:fix**
- **根因:** `URLSession.shared` 默认不信任 server 自签证书,Mac 拒收图片下载。
- **方案:** ImageDownloader 内部建私有 URLSession + TrustAllDelegate(私有工具场景下统一信任 server)。
- **修改文件:** `MacClient/Sources/CCAnywhere/Services/ImageDownloader.swift`

#### 需求 14:InputInjector 图片只发一次回车不发送
- **类型:fix**
- **根因:** Claude TUI 的 file mention 流程是"`@<path>\r` → 文件转 pill,光标停在输入框等待 → 再 `\r` → 真发送"。inject 单次回车只完成第一步。
- **方案:** inject `@<path>\r` 后 sleep 200ms + 再 inject `\r`。
- **修改文件:** `MacClient/Sources/CCAnywhere/Services/InputInjector.swift`

#### 需求 34:Mac 端 Edit 菜单缺失导致 Cmd+V 不工作
- **类型:feat**
- **根因:** AppDelegate.installMainMenu 只装了 App 菜单和 Help 菜单,**没有 Edit 菜单**,macOS 不会自动绑定 Cmd+X/C/V 到 responder chain → SwiftTerm/TextField 收不到。
- **方案:** 补充完整 Edit 菜单(撤销/重做/剪切/拷贝/粘贴/全选,标准 NSText selector)。
- **修改文件:** `MacClient/Sources/CCAnywhere/App/AppDelegate.swift`

#### 需求 33(Mac 端部分):Slash commands 扫描
- **类型:feat**
- **方案:** 新增 `SlashCommandBridge.swift`,扫描 `~/.claude/commands/` + `<workDir>/.claude/commands/` + `~/.claude/plugins/cache/*/.../commands/` + 内置清单,响应 `slash.list.request` 返回 commands 数组。
- **修改文件:** `MacClient/Sources/CCAnywhere/Services/SlashCommandBridge.swift`(新)、`App/DependencyContainer.swift`

### Server 端修复(4 个)

#### 需求 10:Server-driven push tab.list
- **类型:feat**
- **根因:** phone 自行 fetch tab.list 时序敏感(ws 未 connected 时 send 抛错丢失请求 → phone UI 永久 loading)。
- **方案:** phone bind 成功后,如果 Mac 在线,server 主动给 Mac 发 `tab.list.request`,Mac 回 `tab.list.response` 由 RouteFromMac 广播给所有 phone。
- **修改文件:** `Server/internal/server/server.go`

#### 需求 11:Server public_host IP 修正
- **类型:fix**
- **根因:** `config.yaml` 的 `public_host: "192.168.2.115:8443"` 是旧 LAN IP,Mac 实际是 `192.168.2.126`,server 生成的图片 URL 走不通。
- **方案:** 改为 `192.168.2.126:8443`。
- **修改文件:** `local-deploy/config/config.yaml`

#### 需求 16:Server 图片不立即删 + 延迟 GC
- **类型:feat**
- **根因:** `HandleFetched` Mac 下载后立刻删 inbox 文件,phone 端无法预览历史图片。
- **方案:** `pendingUpload` 增加 `Fetched`/`FetchedAt`;`HandleFetched` 只标记不删;`sweepExpired` 区分 unfetched-expired(5min)和 fetched-expired(7天 fetchedTTL)。
- **修改文件:** `Server/internal/image/image.go`

#### 需求 17:Server 新增 image.download.url 协议
- **类型:feat**
- **方案:** 新增 `image.download.url`(phone→server,by upload_id)+ `image.download.url.response`(server→phone,带 fetchedTTL 内有效的签名 URL)。phone 用这个协议请求历史图片预览。
- **修改文件:** `Server/internal/protocol/messages.go`、`Server/internal/server/server.go`、`Server/internal/image/image.go`

#### 需求 33(Server 端部分):slash.list 协议透传
- **类型:feat**
- **方案:** 协议常量 `TypeSlashListRequest/Response`,Route 层 phone→mac 和 mac→phone 各加一个 case。
- **修改文件:** `Server/internal/protocol/messages.go`、`Server/internal/router/router.go`

### Android 端修复(23 个)

#### 需求 3:trustSelfSigned 默认 true + 设置项
- **类型:fix**
- **根因:** Phone 端默认拒绝自签证书,TLS 握手 EOF。
- **方案:** ServerConfig 默认 true、ManualInputScreen `_trust` 默认 true、tryParseQr 默认 true;Settings 页加开关(改后写 secure_storage + 重连 WS)。
- **修改文件:** `models/server_config.dart`、`features/auth/manual_input_screen.dart`、`features/settings/settings_screen.dart`

#### 需求 4 + 8 + 22 + 30 + 32:消息解析持续完善
- **类型:fix**
- **根因:** Claude JSONL 真实结构是 `{type:"user|assistant", uuid, message:{role, content:[...]}, ...}`(双层嵌套),且含大量元数据 type。
- **演进:**
  - 需求 4:支持嵌套 message.content + 跳过 `attachment/system/summary/file-history-snapshot`
  - 需求 8:跳过 `isMeta=true` user / 单条 `No response requested.` assistant / `bridge-session`
  - 需求 22:跳过 `permission-mode / last-prompt` + 兜底:含 sessionId 无 message 的行视为元数据
  - 需求 30:`message.content` 是 string 时也调 `_rewriteInboxRefAsAttachment`(string 分支之前漏掉)
  - 需求 32:`<command-name>...</command-name>` XML 是 Claude 内部 slash command transcript,过滤
- **修改文件:** `models/message.dart`

#### 需求 5:文字"发送中"不消失
- **类型:fix**
- **根因:** `input.text` 协议是 fire-and-forget,但 `sendText` 没在 send 成功后清 `isLocalPending`,导致 UI 永远显示"发送中"。
- **方案:** await send 后立刻清 pending。
- **修改文件:** `data/chat_repository.dart`

#### 需求 6:router redirect 时序导致首次启动停在 onboarding
- **类型:fix**
- **根因:** `initialConfigProvider` 异步加载未完成时 redirect 返回 null,GoRouter 不会重跑 redirect。
- **方案:** ValueNotifier + ref.listen(initialConfigProvider) 触发 GoRouter refreshListenable,provider resolve 后重跑 redirect。
- **修改文件:** `routes/app_router.dart`

#### 需求 9:tab list 永远 loading
- **类型:fix**
- **根因:** `_TabListScreenState.initState` 时 ws 可能还在 connecting,send tab.list.request 抛 NotConnectedException 被静默吞,然后再也不重试。`tabsStream` 是 broadcast 无 replay 价值。
- **方案:** TabRepository 订阅 ws stateStream,connected 时自动 requestList;构造时如果 ws 已 connected 立即触发一次。
- **修改文件:** `data/tab_repository.dart`

#### 需求 12:发送瞬间立刻清输入框 + 图片预览
- **类型:fix**
- **方案:** `_send()` 进入即 `setState(() { _pending.clear(); _busy = true; })` + `_ctrl.clear()`,失败由 chat_repository 在对话区的 attachment 卡片承接。
- **修改文件:** `features/chat/widgets/input_bar.dart`

#### 需求 15:tool_use 卡片永远"待批准"
- **类型:fix**
- **根因:** JSONL 中 tool_use 行不带 status,phone 端默认 pending。tool_result 行带 `tool_use_id` 关联,但 phone 端没扫描合并。
- **方案:** `_mergeMessages` 扫描 toolResult,把对应 toolUse 升级为 executed。
- **修改文件:** `data/chat_repository.dart`

#### 需求 18 + 19:filename → upload_id 持久化映射
- **类型:feat**
- **方案:** 新增 `ImageRefStore`(基于 SharedPreferences),sendTextWithImages 上传成功后 `store.put(filename, upload_id)`。
- **修改文件:** `data/image_ref_store.dart`(新)、`data/chat_repository.dart`、`data/image_upload_service.dart`(upload 返回 uploadId)

#### 需求 20 + 21 + 29:`@<path>` 改写为 attachment + 缩略图渲染
- **类型:feat**
- **方案:**
  - message.dart `_rewriteInboxRefAsAttachment` 识别 inbox path,改写为 MessageKind.attachment(regex 允许 path 含空格如 `Application Support`,但 inbox filename 段不含空格)
  - chat_repository 检测到 attachment 无 remoteUrl + filename 命中 ImageRefStore → 发 `image.download.url`,收到 response 后 update remoteUrl
  - attachment_card 优先本地 Image.file → fallback Image.network → loading/error 状态
  - main.dart 加 `HttpOverrides.global` 全局信任自签(让 Image.network 也能从 server 下载)
- **修改文件:** `models/message.dart`、`data/chat_repository.dart`、`features/chat/widgets/attachment_card.dart`、`main.dart`、`models/protocol_message.dart`

#### 需求 23:文字双气泡 dedup

**核心担忧**：`message_card_list.dart:296` 对 `toolName == 'TodoWrite'` 一刀切消音，是否会误杀 SubAgentFoldedBlock 内的子 agent TodoWrite 显示？

**独立核实结论：**不会误杀。

**论证链**：

1. SubAgentFoldedBlock 渲染路径（`sub_agent_folded_block.dart`）：
   - 入口：MessageCardList 内 `MessageKind.subAgentBlock` 分支 → 取 ChatRepository._subAgentBlocks → 构造 SubAgentFoldedBlock widget
   - 内部 `_buildChildrenList`（line 229-237）→ 遍历 `block.children`（**raw JSONL Map**，不是 Message） → 每条构造 `_ChildLine` widget
   - `_ChildLine._describeRecord`（line 369-388）case 'tool_use' → 输出 **字符串预览** `'tool_use: $name${hint.isEmpty ? '' : ' · $hint'}'`，由 `Text` widget 渲染

2. **关键事实**：SubAgentFoldedBlock 内部**完全不调用 ToolUseCard**，也**完全不调用 `message_card_list._buildCard`**。它从 raw JSONL Map 直接构造一个 `_ChildLine` widget（icon + text 二元组），自己 layout。

3. 因此 `message_card_list.dart:296` 的消音判断（针对的是 `MessageKind.toolUse` 类型的 Message，而非 raw JSONL）**根本到不了** sub-agent 折叠块内的渲染路径。

4. sub-agent 内 TodoWrite raw JSONL（含 `isSidechain=true` + `tool_use` + `name='TodoWrite'`）的生命周期：
   - 进 `_preprocessSubAgent` → sidechain 分支 line 590-616 → 写入 `SubAgentBlock.children` → **never** 进 passthrough → **never** 在主消息流出现
   - 折叠块展开 → `_ChildLine` 输出 "tool_use: TodoWrite · ..." 字符串

5. 这与变更说明 R-T1-007 的文字描述（"仍走 ToolUseCard 渲染"）**字面不一致**但**功能一致**："不进主 panel + 在折叠块内可见"两个核心需求达成。文档措辞不精准（R-T1-007 说"走 ToolUseCard"，实际是折叠块内的 _ChildLine 字符串预览），但不是代码缺陷。

**结论**：维度 G-26 关注的误杀风险**不存在**。代码行为正确，文档措辞偏差留为 🟢 信息性（详见 🟢-7）。

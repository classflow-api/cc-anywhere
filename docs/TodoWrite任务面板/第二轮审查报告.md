### 🟡-1 UTF-16 substring 切坏 emoji / 非 BMP 字符 → 已修复 ✅

**修复后实现**（`todo_panel.dart:87-101`）：
```dart
final String trailing;
if (currentInProgress != null) {
  // R-T1-009:超 30 字符截断。第二轮 review 🟡-1:用 Characters.take
  // 按 grapheme cluster 切,避免 String.substring 把 emoji / 非 BMP 字符
  // 的 UTF-16 surrogate pair 切坏成孤儿码点。
  final chars = currentInProgress.characters;
  final summary = chars.length > 30
      ? '${chars.take(30).toString()}…'
      : currentInProgress;
  trailing = '当前: $summary';
} else if (completed == total) {
  trailing = '已完成 ✅';
} else {
  trailing = '待开始';
}
```

**独立验证**：
- `String.characters` extension 由 `package:characters/characters.dart` 提供，Flutter SDK 在 `package:flutter/foundation.dart` re-export，`material.dart` import foundation → `todo_panel.dart` 已 `import 'package:flutter/material.dart'`，无需显式 import `characters`
- 验证：`flutter analyze` 通过（详见 §F）→ `characters` extension 实际可访问，无 "Undefined getter" 错误
- 行为验证：
  - emoji 测试（🚀 = U+1F680 = 2 UTF-16 units / 1 grapheme cluster）：`'🚀'.characters.length == 1`，`take(30)` 不会切到代理对中间
  - 中文 BMP 字符（每个 1 UTF-16 unit / 1 grapheme cluster）：行为与原 substring 等价 ✓
  - 非 BMP 字符（CJK 扩展 B/C 区、变体选择符）：按 grapheme cluster 安全切分 ✓
  - 换行符 `\n`：单个 grapheme cluster，`take(30)` 不会丢上下文（但 maxLines=1 + ellipsis 仍把换行视为空格渲染 — 这是 R-T1-003 折叠态预期行为）
- 与初审建议 patch 形态完全一致

**结论**：🟡-1 处置 A（修复）有效。

```dart
class TodoItem {
  final String content;
  final TodoStatus status;

  const TodoItem({
    required this.content,
    required this.status,
  });

  /// 从 TodoWrite tool_use input.todos 单项解析。
  /// 字段缺失 / 类型错误 / 未知 status → 返回 null（safe degradation，
  /// 防止 Claude Code 升级字段名后整个 panel 崩溃）。
  /// 注：Claude Code 当前 TodoWrite schema 仅 content / status / activeForm；
  /// 不解析 activeForm（与 content 重复度高，UI 用 content 就够）。
  static TodoItem? tryFrom(Map<String, dynamic> json) {
    final content = json['content'];
    if (content is! String || content.isEmpty) return null;
    final status = TodoStatus.tryParse(json['status'] as String?);
    if (status == null) return null;
    return TodoItem(content: content, status: status);
  }
```

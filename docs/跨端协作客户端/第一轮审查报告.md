### 3. **关键 payload 字段类型不一致（严重）**

#### sub_token_id 类型在三端定义不一致

| 字段 | Server (Go) | MacClient (Swift) | 后果 |
|------|-------------|-------------------|------|
| `DeviceBound.sub_token_id` | `int64` (JSON number) | `DeviceBoundPayload.subTokenId: String` | Mac 解码失败 → `device.bound` 事件被静默丢弃，新绑定设备**不会出现**在 Mac 设备列表 |
| `DeviceListResponse.devices[i].id` | `int64` | Mac 读 `d["id"]?.asString` → nil → 退回为随机 UUID | 每次刷新设备列表，行 id 都不同；UI 看起来正常但**撤销按钮失效** |
| `DeviceRevoked.sub_token_id` | `int64` | Mac 读 `dict["sub_token_id"]?.asString` → nil | 撤销响应到达后 Mac UI 不会从列表移除该行 |
| `DeviceRevoke.sub_token_id` (Mac→Server) | `int64` | `DeviceRevokeRequest.subTokenId: String` | Server `env.DecodeData(&req)` 把 string 解码为 int64 报错 → Mac 端「撤销」按钮**完全无效** |

#### `input.image` payload 缺少 upload_id

| 字段 | Server 发送 | Mac 接收模型 |

| `tab_id` | ✓ | ✓ |
| `image_url` | ✓ | ✓ |
| `filename` | ✓ | ✓ |
| `sha256` | ✓ | ✓ |
| **`upload_id`** | ✓ | **缺失** |

#### `image.fetched` payload

| 字段 | Server 期望 | Mac 实际发送 |

### 4. 协议 envelope 差异

| 字段 | Server | Mac | Android |
|------|--------|-----|---------|
| `type` | ✓ | ✓ | ✓ |
| `id` | ✓ | ✓ | ✓ |
| `ts` (ISO8601 时间戳) | 服务端发送时包含 | **Mac envelope 无 ts 字段** | **Android envelope 无 ts 字段** |
| `data` | `json.RawMessage` | `AnyJSON?` | `Map<String,dynamic>` |

`ts` 缺失不影响功能（消费方不依赖），但与技术文档 §6.1 envelope 规范不符。

### 5. WebSocket URL 路径不一致

| 端 | URL 构造 | 路径 |
|----|---------|------|
| Server (`server.go:84`) | `mux.HandleFunc("/ws", ...)` | `/ws` |
| MacClient (`ServerConfig.swift:36`) | `wss://\(server):\(port)/` | `/` |
| AndroidClient (`ws_client.dart:138`) | `wss://${server}:${port}/ws` | `/ws` |

**MacClient 永远连不上 Server**，因为路径 `/` 没有任何 handler 注册，Server 会返回 404 而非升级 WebSocket。

- [ ] 通过：未发现阻塞问题

### 核心阻塞综述

整套代码各端分别均有较完整结构，单端跑通基本功能（创建 Tab、启动 claude、监听 JSONL、Android 绑定）。但是**三端协议契约与端到端连通存在严重缺陷**，导致以下端到端场景**完全不可工作**：

1. **MacClient 永远连不上 Server**（URL 路径 `/` vs `/ws`）
2. **手机绑定后 Mac 不会显示新设备**（`device.bound.sub_token_id` 类型不匹配）
3. **撤销设备按钮无效**（`device.revoke.sub_token_id` 类型不匹配；`device.list.response.id` 解析丢失）
4. **手机端 Tab 列表永远是空**（Mac 不主动发 tab.list，不响应 tab.list.request）
5. **手机端进 Tab 看不到任何历史消息**（Mac 不响应 msg.history.request）
6. **图片中转后 Server 临时文件不会立即删除**（Mac 发的 image.fetched 缺 upload_id），违反 R-M7-04 / R-S6-04 的"立即删除"约定

修复策略以"协议契约对齐"为第一优先级，建议在第一轮整改中统一处理类型一致性（int64 vs String）与 envelope 字段（ts/upload_id）。第二优先级补齐 MacClient 协议响应端（tab.list / msg.history / tab.changed）。

### 6.1 选择 🛑 的理由

1. **0→1 项目 + 从未真机运行**：三端代码已通过编译与静态检查（`go build/vet/gofmt`、`swift build` 0 warning、`flutter analyze` No issues），但**从未在真实 macOS 14+ / Android 10+ / VPS 上跑过端到端**。这是最大的不确定因素。
2. **跨端协议契约只在 review 中验证**：三轮 review 修复了 13 个 🔴 阻塞，其中多个是契约不一致（URL 路径、sub_token_id 类型、upload_id 透传、tab.list / msg.history 双向链路）。代码层面已对齐，但**真机首次跑通才能确认所有 envelope/payload 字段端到端无新 bug**。
3. **无任何自动化测试覆盖**：三端业务侧 0 单元测试 + 0 集成测试 + 0 端到端测试，回归保护完全依赖人工。
4. **大量已知 🟡 警告**：单条不致命，但累计 24+ 个未修复警告，叠加在"零真机验证"上风险被放大。
5. **个人工具风险容忍度高 ≠ 零验证**：项目无支付 / 无外部 API / 无多租户，确实可以省略形式化测试套件；但 0→1 项目最基础的"端到端跑通"验证不可省略。

### 6.2 必须完成的人工测试清单

见第 5 节 18 项。**核心 P0 子集**（≤ 1 小时）：

- Server 部署 + healthz 通

# TodoWrite 任务面板 - 变更说明

> 文档版本：v1.0
> 级别：L3 - 小型变更
> 范围：仅 Android 端 UI 改造（mac 零改动 / 协议零改动 / Server 零改动）

## 1. 变更内容

### 1.1 现状

Claude Code 内置的 `TodoWrite` 工具用于实时维护任务清单（pending / in_progress / completed 三态）。Mac TUI 上 Claude 会渲染为顶部固定 "Update Todos" panel 实时更新。

手机端目前的表现：

- TodoWrite tool_use **已经通过 msg.stream 透传**（dumb proxy 红利，零协议改动）
- 但手机端**没有专属渲染** — 走通用 `ToolUseCard`，显示为黄色边框 + "TodoWrite · 待批准" + JSON 参数
- Claude 在一次会话里通常 TodoWrite **十几次到几十次**（每完成一项就重写整个 todos 列表）→ 手机聊天流被淹没，看不到当前清单状态

### 1.2 改造后

新增**顶部固定 panel**（方案 C）：

```
┌─ Tab 顶部（在终端区上方） ────────────────────┐
│ 📋 任务进度  3/7 · 当前: 实现 X          [▼]│ ← 折叠态
└──────────────────────────────────────────────┘

点击 [▼] 展开后：
┌──────────────────────────────────────────────┐
│ 📋 任务进度  3/7 · 当前: 实现 X          [▲]│
├──────────────────────────────────────────────┤
│  ✅  写 PRD                                  │
│  ✅  写技术文档                              │
│  ✅  实现 Service                            │
│  🔄  实现 X（当前进行中）                    │
│  ⚪  实现 Y                                  │
│  ⚪  实现 Z                                  │
│  ⚪  跑测试                                  │
└──────────────────────────────────────────────┘
```

**聊天流**：TodoWrite 的 ToolUseCard **消音**（不渲染），避免重复显示。

### 1.3 具体改动点

| 改动 | 现状 | 改造后 |
|---|---|---|
| TodoWrite 显示 | 通用 ToolUseCard（黄色边框 + JSON）| Tab 顶部固定 panel |
| Panel 折叠态 | 无 | "📋 N/M · 当前: <in_progress 任务>" 单行 |
| Panel 展开态 | 无 | 完整 todos 列表 + 三态 icon（⚪ pending / 🔄 in_progress / ✅ completed） |
| 聊天流 TodoWrite 卡片 | 显示通用 ToolUseCard | **消音不显示** |
| 状态管理 | 无 | ChatRepository 维护 sessionId → todos[] 的最新状态 |
| 子 agent 内 TodoWrite | 走通用 ToolUseCard | **不进主 panel**（仍在子 agent 折叠块内显示） |

## 2. 变更原因

### 2.1 用户痛点

- 手机端用户的核心需求：「Claude 现在在干什么 / 整体进度到哪里」
- 当前手机端虽然能透传 TodoWrite 数据，但**无法直观看到当前进度**
- TodoWrite 高频更新（一次会话 10+ 次）淹没聊天流，体验差

### 2.2 设计原则

- **mirror Mac TUI 体验**：Mac TUI 上 TodoWrite 也是顶部固定 panel，手机端对齐
- **单一信息源**：todos 全局状态 ≠ 消息流卡片，避免双重渲染
- **零协议改动**：Claude Code 已经把 TodoWrite tool_use 落进 session.jsonl，msg.stream 已透传，纯手机端 UI 改造
- **子 agent 隔离**：sub-agent 内部 TodoWrite 不影响主 panel（仍归属子 agent 折叠块）

## 3. 影响范围

### 3.1 涉及文件

| 文件 | 改动类型 | 说明 |
|---|---|---|
| `AndroidClient/lib/data/chat_repository.dart` | 修改 | 维护 `Map<TabId, List<TodoItem>> _currentTodos`，msg.stream 处理路径识别 `isSidechain=false` 的 TodoWrite tool_use 时更新 |
| `AndroidClient/lib/features/chat/widgets/todo_panel.dart` | **新增** | 折叠/展开 panel widget，三态 icon，listen ChatRepository todos stream |
| `AndroidClient/lib/features/chat/widgets/message_card_list.dart` | 修改 | `isSidechain=false && toolName=TodoWrite` 的 tool_use 消音（跳过 ToolUseCard 渲染） |
| `AndroidClient/lib/features/chat/chat_screen.dart` | 修改 | 在终端区上方嵌入 TodoPanel widget |

### 3.2 不动的部分

- ❌ 不改 Mac 端
- ❌ 不改 Server
- ❌ 不改协议（msg.stream / TodoWrite tool_use 透传机制不变）
- ❌ 不改 hook bridge
- ❌ 不动 sub-agent 折叠块逻辑（sub-agent 内 TodoWrite 仍走折叠块内 tool_use 渲染）
- ❌ 不动 ask 卡片 / tool_approval 路径
- ❌ 不动其他 tool_use 卡片渲染（仅 TodoWrite 一种被消音）

### 3.3 兼容性

- **数据兼容**：TodoWrite tool_use 协议字段无变化，旧版 mac 也是同样输出
- **多 Tab**：todos 状态按 tabId 维度隔离（A Tab 的 todos 不影响 B Tab）
- **离线/重连**：历史回放含 TodoWrite tool_use 时，ChatRepository 同样路径处理 → panel 显示最后一次 TodoWrite 的状态（与实时一致）

## 4. 验证方式

### 4.1 功能验证

| 场景 | 验证步骤 | 预期 |
|---|---|---|
| 单次 TodoWrite | mac 触发 1 次 TodoWrite（含 3 项 todos） | 手机 panel 顶部出现，"📋 0/3"；展开看到 3 条 ⚪ pending |
| 状态更新 | mac 触发第 2 次 TodoWrite（其中 1 项变 in_progress） | panel 折叠态："📋 0/3 · 当前: <in_progress 任务名>"；展开 1 条 🔄 |
| 任务完成 | mac 触发第 3 次 TodoWrite（in_progress 变 completed + 下一项变 in_progress） | panel "📋 1/3 · 当前: <新 in_progress>"；展开 1 ✅ + 1 🔄 + 1 ⚪ |
| 全部完成 | 所有 todos 变 completed | panel "📋 3/3 · 已完成 ✅"；展开 3 ✅ |
| 聊天流消音 | 在任意 TodoWrite 触发后查看聊天流 | TodoWrite 的 ToolUseCard **不显示** |
| 其他工具不受影响 | Claude 调用 Bash / Read | 其他工具的 ToolUseCard 正常显示 |
| 多 Tab 隔离 | Tab A 跑 TodoWrite，切到 Tab B | Tab B 顶部无 panel 或显示 Tab B 自己的 todos；切回 Tab A 看到 A 的 todos |
| 子 agent 内 TodoWrite | 子 agent 内部触发 TodoWrite | 主 panel **不更新**；子 agent 折叠块内显示 ToolUseCard |
| 历史回放 | 手机重连拉历史含 TodoWrite | panel 显示最后一次 TodoWrite 的状态 |
| 折叠/展开 | 点击 panel 折叠/展开 | 动画 ≤ 200ms，展开内容不阻挡终端区 |

### 4.2 回归验证

- 其他工具的 ToolUseCard 显示正常
- AskQuestionCardRealtime（翻页 + chip + 队列）不受影响
- SubAgentFoldedBlock（子 agent 折叠块）不受影响
- 暗色 / 亮色主题适配

## 5. 业务规则

| 编号 | 规则描述 |
|---|---|
| R-T1-001 | 仅 `isSidechain=false`（主 session 而非子 agent）的 TodoWrite tool_use 进入主 panel |
| R-T1-002 | Panel 按 tabId 维度隔离，每个 Tab 维护自己的 todos 状态 |
| R-T1-003 | Panel 折叠态文字格式：「📋 任务进度 {completed}/{total} · 当前: {in_progress 任务摘要 30 字}」；无 in_progress 时显示「· 已完成 ✅」或「· 待开始」（仅 pending） |
| R-T1-004 | Panel 展开态每项 icon：pending=⚪（圆圈）/ in_progress=🔄（动画转圈）/ completed=✅（绿色对勾） |
| R-T1-005 | 同一 Tab 内连续多次 TodoWrite 仅保留最新状态（覆盖式更新）；不保留历史 |
| R-T1-006 | 聊天流中 `isSidechain=false && tool_name="TodoWrite"` 的 tool_use 消音不渲染 ToolUseCard |
| R-T1-007 | sub-agent 内（isSidechain=true）的 TodoWrite tool_use 仍走 ToolUseCard 渲染（在 sub-agent 折叠块内显示），不进主 panel |
| R-T1-008 | 单题 todos 数 = 0 时不显示 panel（隐藏整个 widget，不占空间） |
| R-T1-009 | 单题摘要超过 30 字符时用 `…` 截断 |
| R-T1-010 | Panel 折叠/展开动画 ≤ 200ms；动画期间不阻塞用户其他操作 |
| R-T1-011 | 历史回放（msg.history.response）路径同样处理 TodoWrite，重连后 panel 显示最后状态 |

## 6. 风险评估

| 风险 | 严重度 | 缓解 |
|---|---|---|
| TodoWrite input 字段 schema 变化（如 Claude Code 升级改了 todos 字段名） | 低 | 解析时 try-catch，字段缺失则 panel 不更新（safe degradation） |
| Panel 占用终端区上方空间，影响多 Tab 切换性能 | 低 | 仅在 todos 非空时显示；折叠态高度 ≤ 40px |
| 多次 TodoWrite 触发频繁重绘 | 低 | ChangeNotifier 仅在 todos diff 时通知；ListenableBuilder 局部 rebuild |
| 子 agent 折叠块内显示 TodoWrite ToolUseCard 跟主 panel 不一致（认知冲突） | 低 | R-T1-007 + R-T1-001 明确隔离；UI 不显式标"子 agent 不进主 panel"，依赖语义自洽 |
| 历史回放过程中 todos 状态频繁切换造成 panel 闪烁 | 中 | 历史回放时 batch 处理（一次 msg.history.response 内只取最后一次 TodoWrite） |

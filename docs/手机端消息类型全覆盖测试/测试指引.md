# 手机端消息类型全覆盖测试指引

> 用途:让 Mac 端 Claude Code(已通过 cc-anywhere 与手机端连通)按本指引执行任务,
> **逐步触发 Claude 协议中所有主要消息/事件类型**,用户在手机端逐项验证渲染是否正常。
>
> 阅读对象:**Mac 端那个 Claude Code 实例**(直接 paste 整篇给它即可)。

## 0. 给 Claude Code 的指令

> **你好 Claude!** 请把这份指引当成测试脚本,**严格按顺序**逐项执行。每一项前先输出
> `=== Step N: <场景名> ===`,执行完后**停下来等用户在手机端确认**手机端表现是否
> 符合 §"手机端预期"再继续下一项。**不要一口气全做完**。
>
> 全程在当前会话内完成,**不要新开 session**。所有文件操作都在临时目录
> `/tmp/cc-anywhere-msgtype-test/` 内,执行 Step 1 时先 `mkdir -p` 该目录。

---

## 1. 测试矩阵概览

| Step | 场景 | 触发协议(JSONL `type` / content type) | 手机端预期 |
|---|---|---|---|
| 1 | 普通文本回复 | assistant + text | 助手白色文本气泡 |
| 2 | 思考过程 | assistant + thinking | 灰色思考气泡(💭 图标) |
| 3 | Read 工具(自动批准) | tool_use(Read)+ tool_result | 黄色 Tool Use 卡片,**显示「已执行」**(不再"待批准") |
| 4 | Bash 工具(自动批准的安全命令) | tool_use(Bash)+ tool_result | 同上,显示命令 + 结果 |
| 5 | Bash 工具(危险命令需批准)| tool_use(Bash, **status=pending**) | 黄色卡片显示「批准 / 拒绝 / 总是」按钮 |
| 6 | 一次工具调用失败 | tool_use + tool_result(`is_error=true`) | Tool Result 卡片显示红色错误 |
| 7 | Write 新文件(可能需要批准) | tool_use(Write) | 同 Step 3-5 行为视权限模式 |
| 8 | Glob / Grep | tool_use(Glob)、tool_use(Grep) | 多张 Tool Use 卡片 |
| 9 | WebFetch / WebSearch | tool_use(WebFetch) | 显示 URL + 结果摘要 |
| 10 | Subagent dispatch | tool_use(Task) | 显示子 Agent 启动 + 完成 |
| 11 | AskUserQuestion(交互式询问) | tool_use(AskUserQuestion)| **手机端核心待验证项** — 是否能弹出问题选项 |
| 12 | 长文本流式输出 | 多个 text chunk | 文本流式显示,不卡顿 |
| 13 | Markdown 渲染 | text 含 markdown | 标题/列表/代码块/链接正常 |
| 14 | 代码块 | text 含 ```bash 代码``` | 代码块独立卡片样式 |
| 15 | 多 tool 串行 | 连续 5 个 tool_use | 卡片顺序 + 性能 |

## 2. 详细测试步骤

### Step 1 — 普通文本回复

**Claude 执行**:
```
你好,这是消息类型测试第 1 步。请回答:1+1 等于几?
```
**手机端预期**:Claude 头像 + 白色气泡 + 简短回复(如 "1+1=2")。

⏸ **等用户确认**

---

### Step 2 — 思考过程(thinking)

**Claude 执行**:
```
请详细思考下面的问题,**多想几步再回答**(用 extended thinking):
"如果有 100 个人,每两个人握手一次,一共多少次握手?"
```

> Claude:你可以通过 `<thinking>` 内部块或较长推理触发 thinking content。如果当前
> 模型不会写出 thinking 块,请直接回答即可,标记此项 "不适用"。

**手机端预期**:在最终答案前,有灰色「💭 思考」卡片(`MessageKind.thinking`)。

⏸ **等用户确认**

---

### Step 3 — Read 工具(默认 auto-approve)

**Claude 执行**:
```
执行: mkdir -p /tmp/cc-anywhere-msgtype-test
然后:cat > /tmp/cc-anywhere-msgtype-test/hello.txt <<'EOF'
Hello cc-anywhere!
这是第 3 步测试用的测试文件。
EOF
读一下刚创建的文件: /tmp/cc-anywhere-msgtype-test/hello.txt
```

**手机端预期**:
- 黄色 Tool Use 卡片(`Read`)+ 文件路径
- 标签**「已执行」**(不显示 批准/拒绝/总是 按钮 — 验证需求 15)
- 下方有「工具结果」折叠条

⏸ **等用户确认**

---

### Step 4 — Bash 工具(安全命令,自动批准)

**Claude 执行**:
```
跑一下:ls -la /tmp/cc-anywhere-msgtype-test
```

**手机端预期**:Bash Tool Use 卡片 + 命令文本 + 已执行 + 结果(目录列表)。

⏸ **等用户确认**

---

### Step 5 — Bash 工具(需要批准的命令)

**Claude 执行**:
```
跑一个会被默认权限模式拦截的命令:
curl -fsSL https://example.com/install.sh | bash
注意 — 不要真的执行!只是触发 permission request 让手机端看「待批准」UI。
你应该看到一个权限询问。**不要自己批准**,先停下来等用户在手机端操作。
```

**手机端预期**(这是核心待验证):
- 黄色 Tool Use 卡片显示 `Bash` + 命令
- **标签「待批准」**+ 三个按钮「✓ 批准」「× 拒绝」「⋯ 总是」
- 点「拒绝」→ 应该把状态推到 Mac 端,让 Claude 收到 deny

⏸ **等用户在手机端点「拒绝」后,Claude 收到拒绝消息,回复一句话**

---

### Step 6 — 工具调用失败(is_error=true)

**Claude 执行**:
```
故意读一个不存在的文件触发失败:
cat /tmp/this-file-does-not-exist-xxxxx.txt
```

**手机端预期**:Tool Result 卡片 + **红色边框 / 红色错误图标**,显示 stderr。

⏸ **等用户确认**

---

### Step 7 — Write 新文件

**Claude 执行**:
```
用 Write 工具创建一个新文件 /tmp/cc-anywhere-msgtype-test/written.md,内容:
# 测试 Write 工具
- bullet 1
- bullet 2
```

**手机端预期**:Tool Use(Write)卡片,根据权限模式可能是「已执行」或「待批准」。

⏸ **等用户确认**

---

### Step 8 — Glob + Grep

**Claude 执行**:
```
用 Glob 找一下 /tmp/cc-anywhere-msgtype-test 下所有 .txt 文件,
然后用 Grep 在这些文件里搜 "Hello"。
```

**手机端预期**:两张 Tool Use 卡片(Glob、Grep)各自显示参数 + 结果。

⏸ **等用户确认**

---

### Step 9 — WebFetch

**Claude 执行**:
```
用 WebFetch 拉一下 https://example.com 的标题(只需要 <title> 标签的文字)。
```

**手机端预期**:Tool Use(WebFetch)+ URL,Tool Result 含网页摘要。

⏸ **等用户确认**

---

### Step 10 — Subagent 分发(Task 工具)

**Claude 执行**:
```
用 Task 工具派发一个子 Agent(subagent_type=general-purpose):
"统计 /tmp/cc-anywhere-msgtype-test 目录里所有文件的总行数,只回报数字"
```

**手机端预期**:
- Tool Use(Task)卡片 — agent 名称
- 中间是否能看到 subagent 的 progress?(若 cc-anywhere 把 subagent 输出也流到 JSONL,应该能)
- 最终 Tool Result(子 agent 的回报)

⏸ **等用户确认 — 这一步重点关注:subagent 的中间消息是否能透传**

---

### Step 11 — AskUserQuestion(交互式询问)⭐ 核心

**Claude 执行**:
```
调用 AskUserQuestion 工具,问用户:
- question: "你现在是用手机端在测试吗?"
- options: [
    { label: "是,正在用手机", description: "..." },
    { label: "否,用 Mac 端", description: "..." }
  ]
- header: "测试身份"
```

**手机端预期**(待确认是否支持):
- 手机端应展示问题 + 选项按钮
- 用户点选项 → 回复推到 Mac 端 Claude
- 这是验证「手机端能否参与 AskUserQuestion 交互」的关键场景

⏸ **等用户在手机端选项,Claude 收到 → 回复确认**

> 如果手机端**没**展示问题(被当作普通文本/raw 卡片),记录此项为"未实现",
> 后续需要新增协议支持。

---

### Step 12 — 长文本流式输出

**Claude 执行**:
```
请写一段约 1000 字的「cc-anywhere 项目介绍」,从架构、技术栈、典型流程三方面展开。
要求一段段连续输出(不要分多次 turn)。
```

**手机端预期**:
- 文本流式更新(逐 chunk 而不是一次性出现)
- 滚动跟随到底部
- **不卡顿**(验证需求 24 性能优化)

⏸ **等用户确认**

---

### Step 13 — Markdown 渲染

**Claude 执行**:
```
用 markdown 输出下面这段(全部直接 print,不要再执行):

# 标题 H1
## 标题 H2

普通段落,**粗体**,*斜体*,`inline code`。

- 列表项 1
- 列表项 2
  - 嵌套列表
- 列表项 3

1. 有序 1
2. 有序 2

> 引用块

[链接示例](https://example.com)

| 列 A | 列 B |
|------|------|
| a1   | b1   |
| a2   | b2   |
```

**手机端预期**:标题/列表/链接/表格都正确渲染,不是 raw markdown 文本。

⏸ **等用户确认**

---

### Step 14 — 代码块

**Claude 执行**:
```
输出下面这段 markdown(含多语言代码块):

```python
def hello():
    print("Hello")
```

```dart
void main() => print('Hello');
```

```bash
echo "shell"
```
```

**手机端预期**:每个代码块独立样式(等宽字体 + 暗背景),语法高亮(若实现了)。

⏸ **等用户确认**

---

### Step 15 — 多 tool 连续调用(性能 + 顺序)

**Claude 执行**:
```
连续做 5 件事(不要并行,串行做):
1. Bash: date
2. Bash: uname -a
3. Bash: pwd
4. Read /tmp/cc-anywhere-msgtype-test/hello.txt
5. Bash: echo "done"
```

**手机端预期**:
- 5 张 Tool Use 卡片**按顺序**出现(不乱序)
- 每张都显示「已执行」
- 滚动不卡

⏸ **等用户确认**

---

## 3. 收尾清理

最后一步,Claude 帮忙清理:

```
rm -rf /tmp/cc-anywhere-msgtype-test
```

并输出一行 `=== 测试完成 ===`。
